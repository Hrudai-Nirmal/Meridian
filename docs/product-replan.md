# ArgusGrid Product Replan

ArgusGrid is now positioned as the AI automation control room for agencies and teams. The early commercial wedge is AI automation agencies that need to prove reliability and ROI to clients.

## Pillars

- Visual automation map: graph-first project maps, PNG export, later share/embed.
- AI value monitoring: cost, quality, latency, tokens, run success, and uptime together.
- Reliable alerting: threshold alerts now; baseline/anomaly alerts next.
- Client-facing reporting: secure report links and exports before full portals.
- Developer instrumentation: open-source Python and JavaScript SDKs as the adoption loop.

## Near-Term Build Order

1. Polish client report links and PNG/PDF exports.
2. Publish SDK preview docs and test against Dify, n8n, GitHub Actions, and OpenAI/custom REST.
3. Add SSE-based live updates for runs, alerts, and node status.
4. Add rolling baseline/anomaly alert mode.
5. Add CSV/read API exports and webhook-out for alert events.

## Explicit Deferrals

- Sankey views are v2+.
- Forecast/correlation views wait until customers have meaningful history.
- SMS alerts are cut until requested by a paying customer.
- Integration templates should stay few and deep before expanding.
- Full client portals come after share links prove demand.
